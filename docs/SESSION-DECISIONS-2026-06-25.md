## 1. UI: "AI 데모앱 향" → 현대 OEM 콕핏 톤
**증상**: 둥근 카드+소프트섀도, 마케팅 히어로, 상태 알약+글자 토글, 이모지(🔒👍👎), 점선 박스,
브라우저 기본 파일 인풋 — 전형적인 "AI가 만든" 시그니처.

**결정 (globals.css 토큰 + 컴포넌트)**:
- 둥글기 12px→**4px**, 소프트섀도 **제거**(깊이는 1px 경계선으로), 그라데이션→솔리드(로고·바차트).
- 마케팅 히어로(대문자 kicker+30px) → **타이트한 콕핏 헤더**(좌측 네이비 룰).
- 연결 상태: 시끄러운 알약 → **작은 점**(정상=조용, 끊김만 빨강+텍스트). 테마: "다크" 글자 → **해/달 아이콘**.
- 이모지 전멸: 🔒→`PII N` 태그, 🎤🔊→텍스트, 👍👎→thumb SVG, ☐→CSS 체크박스.
- 점선 테두리→솔리드, 빈 상태 박스+장식 아이콘 → **담백한 한 줄**, 파일 인풋 `::file-selector-button` 스타일링.
- 케이스 행: 장식 색 스파인 제거 → 우선순위 **의미색 모노 칩**(치명=빨강/높음=주황) + 사망·부상 **아웃라인 태그**.

**근거**: 실무 디자이너는 "크롬(장식) 최소화 + 아이콘으로 말하기 + 상태는 절제 + 데이터 우선". 색은
데코가 아니라 **데이터에** 실어야 의미가 산다. 단일 모달리티 대시보드엔 각진·납작·조밀이 OEM답다.

## 2. 차트: 표/바 → 인터랙티브 도넛 (hover 상세)
**요구**: 주요 부위·점검 결과·구성비를 파이로, 슬라이스 hover 시 상세.

**결정**: 재사용 컴포넌트 `Donut.tsx` — SVG `stroke-dasharray` 누적 세그먼트, **hover 시 가운데에
건수·% 표시 + 슬라이스 강조 + 범례 동기화**. `colors` prop으로 의미색 주입(점검 결과: 양호=초록 등).
적용: 분석 주요 부위, 리포트 주요 부품, **점검 결과 분포**(원문 표는 per-item 상세라 유지).

**트레이드오프 (정직)**: 순위 비교는 **바가 도넛보다 정확**(비슷한 슬라이스 비교 난이). 시계열 추세는
파이 부적합 → **바 유지**. 도넛은 구성비·glance용으로만.

## 3. 히어로 캐러셀 + 자동 갱신
- 히어로: refresh마다 한 차종 → **활동 상위 6 차종 자동 슬라이드**(3.8s, 페이드, hover 멈춤, 점 인디케이터).
- 홈: **20초 폴링 + 탭 복귀 시 즉시** 재조회 → 케이스 **해결/추가**가 새로고침 없이 반영. "자동 갱신 HH:MM:SS" 표시.
- 연결 상태: 마운트 1회 → **10초 폴링 + 탭 복귀**. 중간에 백엔드 꺼지면 점이 빨갛게 + "연결 끊김".
- **근거**: "실시간 느낌"의 정공법은 폴링(앱 상태 변화 반영). 백그라운드 탭에선 폴링 skip(낭비 방지).
  데이터 원천(NHTSA) 자체 갱신은 별도 — refetch+`/refresh` 또는 일일 스케줄(향후).

## 4. 트리아지 우선순위에 최신성 가중
**요구**: 심각도만 말고 최신성 비중.

**결정**: `우선순위 = 사망×100 + 부상×10 + 화재×5 + 사고×3 + 최신성`,
`최신성 = ROUND(12·exp(−경과일/180))`(반감 ~180일). 날짜 `MM/DD/YYYY`/YYYYMMDD/ISO 3중 파싱(실패 0).

**근거/검증**: 가중 12라 **사망(100)은 못 넘고**(오래된 치명 케이스가 최근 경미에 안 밀림), 무-심각
케이스끼리는 최신순으로 정렬. 실측(1,429건): 날짜 파싱 100%, 최상위는 2024 사망 케이스(123) 유지. ✓
라벨도 정정 — "전체 심각도순"은 거짓말이었음(실제 우선순위순) → "우선순위순".

## 5. 불만 접수 내용: 전체 텍스트 + 로컬 요약 (compute-once + cache)
**증상**: 접수 내용이 문장 중간에서 끊김 → **데이터가 소스에서 500자 캡**(1,429건 중 790건 ≥495자).
NHTSA API는 전체를 줌. KONA 케이스 정확히 500자에서 잘림 확인.

**결정**:
1. `refetch_nhtsa_complaints.py`로 **전체 summary 재수집**(이미 자르지 않고 저장).
2. **"비싼 연산 1회 + 영속 캐시"(read-through memoization)** 로 한국어 요약:
   - 키 = 접수번호. 영속 저장소(`GeneratedReport` type="SUMMARY") **먼저 조회** → 히트면 즉시.
   - 미스면 `ollama.ask`로 **딱 1회** 요약 → 저장 → 반환. DB 기반이라 재시작 견딤.
   - 엔드포인트 `/api/agent/case-summary`, 프론트는 불만 모달에서 **lazy 로드**.
- **근거**: 전량 1,429건 배치는 노트북(단일 Ollama)을 때림 → **온디맨드로 분산 + 1회만**. 새 테이블
  안 만들고 기존 범용 캐시 재사용(YAGNI). 목록은 원문 미리보기, 상세는 요약+원문.

## 6. 버그 수정: 케이스 리포트 적재(jsr310)
**증상**: `[case-report] 적재 실패: Java 8 date/time ... jackson-datatype-jsr310 ... ["date"]`.
**원인**: `ReportService`의 손수 만든 `ObjectMapper`가 `LocalDate`(out의 "date")를 직렬화 못 함
(Spring 자동 매퍼와 달리 모듈 미등록). **수정**: `mapper`에 `JavaTimeModule` 등록(한 줄). 요약 캐시는
문자열만 저장이라 무관하지만, 케이스 리포트 적재가 이걸로 복구됨.

## 7. 운영: 적재 동시성 · JVM
- **적재**: 권 fan-out 금지(직렬), 처리량은 백엔드 `ingest.embed.concurrency`로. 실측 천장 = **4 OK / 8 crash**.
  (상세 RAG-INGEST-SCALING.md §8.)
- **JVM 크래시**(`JVMDUMP`/`javacore`/`Snap`/`jitdump`) = **IBM Semeru(OpenJ9)** 흔적. HotSpot은 안 만듦.
  → **Temurin 21**로 전환해야 안 죽음(프로젝트 이력의 그 문제 재발).
