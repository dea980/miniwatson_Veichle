## 1. 동작하는 풀 파이프라인 (완성)

```
질문(한국어/음성) → RAG 검색(매뉴얼 근거 grounding)
   → 파인튜닝 vehicle 모델(Qwen2.5-1.5B LoRA, Q4 온디바이스)이 한국어로 생성
   → 출처(sources) + 거버넌스(감사, PII) 로그 → (음성 TTS)
```

- ✅ 데이터 수집: NHTSA 리콜 119건(원본 JSON) + Internet Archive 오너스 매뉴얼(취급설명서) PDF
- ✅ 자동차 RAG: `vehicle` 네임스페이스, 매뉴얼 근거 + **한국어 답변** (영어 코퍼스 → 한국어 강제)
- ✅ 리콜 text-to-SQL: 차종별 집계 (PALISADE 26, SANTA FE 24, ELANTRA 18 …)
- ✅ LoRA 파인튜닝: MLX, M2 로컬, GPU 없이
- ✅ 경량화: GGUF Q4 양자화 → Ollama 서빙 (온디바이스)
- ✅ 거버넌스: 멀티프로바이더 + 감사, PII (H-Chat 정합)
- ✅ 음성: 브라우저 STT/TTS (🎤 질문 / 🔊 듣기)
- ✅ UI: Next.js (7탭) + 모델 교체로 base↔FT 비교, 정적 UI 패리티
- ✅ **캡스톤 멀티툴**: 차종 종합 진단서(리콜+불만+매뉴얼), 이미지진단(Vision+OCR+RAG)→필요부품(샘플견적)
- ✅ 멀티테이블 레지스트리(`recalls`, `complaints`, `parts`) + Agent 동적 테이블 선택 + SQL 자기수정

## 2. 정량 지표

**LoRA 파인튜닝 (Qwen2.5-1.5B, 122 train / 12 val, MLX iters 800)**
- Train loss 3.7 → 0.12, **Val loss 최저 1.54 @ iter 200** → 이후 상승(과적합) → **iter-200 체크포인트 채택**(early stopping).
- LLM-judge (n=12): base 10/12 vs **FT 11/12** (소량이고 약판정이라 노이즈 범위; 정성적으론 도메인 말투가 살아 있고 퇴화 없음).

## 2.1 진단서 종합 — 모델별 벤치 (동일 입력, 모델만 교체)

PALISADE 진단서를 같은 입력으로 모델만 바꿔 생성. **집계(KPI, 차트, 점검표)는 결정적 SQL이라 전 모델에서 숫자가 동일하게 정확**하고, 차이는 **LLM 종합 서술**에서만 났다.

| 모델 | 크기 | 종합 서술 결과 |
|---|---|---|
| `vehicle-qwen2.5-1.5b` (FT) | 986MB (Q4) | 중국어 누수(`枕头囊气囊`), 반복 퇴화(`———`), 오역(hitch→"척수"). 도메인 말투는 살아있으나 다중사실 종합에서 붕괴 |
| `qwen2.5:7b-instruct` | 4.7GB (Q4) | 충실한 종합, 일부 외국어 토큰 누수(`ています`)·경미한 환각. 실사용 가능 |
| `ibm/granite4:latest` | 2.1GB | 깨끗하고 일관됨 (비교군 중 가장 안정) |
| `gemma4` | 9.6GB | **500 에러** — M2에서 너무 무거워 타임아웃/메모리 압박, 온디바이스 부적합 |

판단:
- **1.5B의 붕괴는 용량 한계.** 한국어 다중사실 종합 같은 어려운 작업에서 사전학습의 중국어로 회귀하고 반복 퇴화한다. 파인튜닝(스타일)으로는 안 메워지는 추론/충실도 문제.
- **gemma4(9.6GB)는 크기 자체가 온디바이스 부적합** — 모델 크기 vs 지연·안정성 트레이드오프의 실증.
- **7B가 균형점**(품질·지연·안정). 다음 단계는 그 7B를 도메인 LoRA(QLoRA, CJK 필터 데이터 + 진단서 형식 예시)로 다듬어 외국어 누수까지 제거.
- 핵심: **신뢰가 필요한 수치는 결정적 SQL로 격리**했기에, 약한 모델이 서술을 망쳐도 표·차트·KPI는 항상 정확하다(환각의 폭발 반경 차단).

## 3. 발견과 판단 (엔지니어링 의사결정)

1. **데이터 품질이 9할** — 약한 생성모델(granite4)은 음역오류와 환각("하다모"); qwen2.5:7b distillation + 한자 필터로 해결.
2. **Early stopping** — val loss 모니터링해 과적합 전(iter 200) 체크포인트 채택.
3. **양자화 폭주 제어** — Q4 단독은 반복/환각 → ChatML stop 토큰 + repeat_penalty로 제어.
4. **언어 강제** — 영어 코퍼스 + 한국어 사용자 → 프롬프트로 한국어 출력 강제.
5. **RAG = grounding** — 단독 FT는 환각("자석 안전띠"); RAG로 매뉴얼 근거 주입해 사실 교정.
6. **자원 직렬화** — 단일 M2에서 학습과 추론을 동시에 = 메모리 초과 → 직렬화(운영은 노드 분리 근거).
