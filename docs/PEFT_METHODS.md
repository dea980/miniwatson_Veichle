# 파인튜닝 / PEFT 방식 정리 (레퍼런스)

> 도메인 적응 기법을 "학습량 vs 비용 vs 효과" 축으로 정리. 이 프로젝트의 선택 근거 포함.
> 관련: [WHY_LORA.md](WHY_LORA.md), [VEHICLE_ARCHITECTURE.md](VEHICLE_ARCHITECTURE.md)

## 0. 큰 스펙트럼 — 학습을 얼마나 하나

```
학습 없음 ───────────────────────────────────────────▶ 전부 학습
프롬프트  →  RAG  →  PEFT(LoRA/QLoRA/...)  →  풀 파인튜닝
(가중치 X)  (지식주입)   (소수 파라미터)        (전체 가중치)
싸고 빠름 ◀───────────────────────────────────────▶ 비싸고 강력
```

- **프롬프트/Few-shot**: 가중치 안 건드림. 가장 싸고 빠르나 한계 명확.
- **RAG**: 가중치 안 건드림. **지식**을 질의 시점 주입. (행동/말투는 못 바꿈)
- **PEFT**: 소수 파라미터만 학습 → 행동/도메인 적응을 싸게. ← 이 프로젝트
- **풀 파인튜닝**: 전체 갱신. 최강이나 GPU 농장이 필요하고 망각 위험이 있음.

> 실무 순서: 프롬프트로 안 되면 → RAG → 그래도 부족하면 PEFT → 정말 필요할 때만 풀 FT.

## 1. PEFT 방식 비교

| 방식 | 핵심 아이디어 | 학습 파라미터 | 장점 | 단점/주의 |
|---|---|---|---|---|
| **LoRA** | 가중치 변화 ΔW를 저랭크 A·B로 근사, 그것만 학습 | ~0.1–1% | 표준이고 생태계 풍부, 어댑터 스왑, 망각↓ | rank 튜닝 필요 |
| **QLoRA** | 원본을 4bit 양자화로 올리고 그 위에 LoRA | ~0.1–1% | 메모리 대폭↓(큰 모델도 단일 GPU) | 약간의 정밀도 손실 |
| **Adapter (Houlsby)** | 층 사이에 작은 병목 MLP 모듈 삽입 | ~1–5% | 안정적, 태스크별 모듈화 | 추론 지연 약간↑ |
| **Prefix Tuning** | 각 층 attention에 학습되는 "가상 prefix" 추가 | <1% | 생성 태스크에 강함 | 구현 까다로움 |
| **Prompt Tuning** | 입력 임베딩 앞에 soft prompt 토큰만 학습 | 극소 | 가장 가벼움 | 큰 모델에서만 효과적 |
| **P-Tuning v2** | 모든 층에 prompt 적용(prefix류 일반화) | <1% | 작은 모델서도 안정 | — |
| **IA³** | 활성값을 스케일하는 소수 벡터만 학습 | 극소 | 초경량, 빠름 | 표현력 제한 |

**공통 철학**: 원본 가중치 **동결(freeze)** + **소수 파라미터만** 학습.

## 2. 어떤 걸 언제 쓰나 (결정 가이드)

- **GPU 없음 / 작은 모델(≤3B)** → **LoRA** (이 프로젝트, MLX)
- **큰 모델(7B+)인데 메모리 빠듯** → **QLoRA** (4bit + LoRA)
- **여러 태스크를 한 모델에 깔끔히 분리** → Adapter / 태스크별 LoRA 어댑터 스왑
- **데이터 극소, 초경량 적응** → Prompt Tuning / IA³
- **최고 품질 + 자원 충분 + 도메인 격차 큼** → 풀 파인튜닝 (드묾)

핵심 튜닝 노브 (LoRA 기준):
- `r`(rank): 8~16 일반적. 클수록 표현력↑, 파라미터↑.
- `alpha`: 스케일(보통 2×r).
- `target_modules`: 어느 층에 붙일지(q/k/v/o proj 등).
- `dropout`, `learning_rate`, `iters/epochs`.

## 3. 인접 개념 (PEFT는 아니지만 같이 묶임)

- **SFT (Supervised Fine-Tuning)**: instruction 쌍으로 지도학습. *방법론*이고, 그걸 LoRA로 하면 "LoRA 기반 SFT". (이 프로젝트가 이것)
- **선호 정렬(RLHF / DPO)**: 사람 선호로 정렬. SFT 다음 단계로, 데이터와 비용이 큼.
- **지식 증류(Distillation)**: 큰 선생 모델로 작은 학생 모델 학습. → `build_dataset.py`에서 qwen2.5:7b로 데이터 생성하는 게 이 발상(데이터 distillation).
- **RAG**: 학습이 아니라 검색 주입. 파인튜닝과 **보완** 관계.

## 4. 이 프로젝트의 선택

| 항목 | 선택 | 이유 |
|---|---|---|
| 적응 방식 | **LoRA (PEFT) + RAG 병행** | GPU 없는 M2, 도메인 행동 적응 + 지식 주입 둘 다 |
| 백엔드 | MLX LoRA (맥) / HF peft (Colab) | 로컬 Metal 학습 가능 |
| 데이터 | qwen2.5:7b로 distillation → instruction SFT | 약한 생성모델은 품질 붕괴 → 선생 모델 키움 |
| 다음 후보 | QLoRA(더 큰 base 시), 어댑터 스왑(판매/제조/AS 도메인별) | 확장 시 |

**한 줄**: 자원 제약(맥, GPU 없음) + 목표(도메인 행동 적응)에 가장 맞는 PEFT = **LoRA**. 큰 모델로 가면 QLoRA, 다도메인이면 어댑터 스왑.
