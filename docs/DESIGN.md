# UI 디자인 시스템 & 결정

> 프론트(Next.js)의 디자인 원칙, 토큰, 결정 기록. "AI 데모 느낌"을 줄이고 자동차 도메인 실서비스 톤으로.
> 무프레임워크(Tailwind/shadcn 없이) — CSS 변수 + 컴포넌트만. 빌드 리스크 0.

---

## 1. 방향 — 현대차풍 미니멀 프리미엄

- **딥 네이비 정체성** (`#002C5F`) — 현대 브랜드 톤. 라이트=네이비 버튼, 다크=블루.
- **큰 타이포 위계** — 섹션마다 30px 히어로 타이틀 + 설명. 여백을 *의도된* 프리미엄으로(현대 공홈식).
- **절제** — 과한 색과 그림자 지양, 얇은 보더 + 은은한 섀도.

## 2. "AI 같다"의 원인과 처방

| 증상 | 원인 | 처방 |
|---|---|---|
| 제네릭함 | 기본 폰트 | **Pretendard** 한글 웹폰트 |
| 장난감 느낌 | 이모지 아이콘 | **SVG 라인 아이콘**(사이드바, 헤더) |
| 휑함 | 작은 폼 + 흰 여백 | **히어로 타이틀 + 설명 + 푸터**로 채움 |
| 밋밋 | 약한 정체성 | 네이비 브랜드색 + 레터마크 로고 + 좌측 액센트 네비 |

## 3. 디자인 토큰 (CSS 변수, 라이트/다크)

- 색: `--accent`(네이비 #002C5F), `--accent-2`(액션 블루), `--bg/surface/surface-2`, `--border`, `--text/muted`, `--ok/danger`.
- 라운드: `--radius: 12px`(카드), 버튼 6px(현대풍 각짐).
- 섀도: 2단 은은한 섀도. 다크는 강도↑.
- 다크모드: `[data-theme="dark"]`로 토큰 오버라이드, 토글은 `localStorage` 저장.

## 4. 레이아웃

```
[사이드바] 브랜드(레터마크) + 라인아이콘 네비(좌측 액센트 바)
[콘텐츠]  얇은 breadcrumb 탑바(+다크토글)
          히어로(kicker + 30px 타이틀 + 설명)
          패널들(카드)
          푸터(브랜드 라인)
```
- 반응형: ≤760px에서 사이드바 → 가로 내비.
- 콘텐츠 max-width 1040px로 가독성 유지.

## 5. 컴포넌트 규칙

- **카드**: 흰 표면 + 얇은 보더 + 섀도 + 라운드. 섹션 제목 16px.
- **버튼**: primary=네이비 솔드, ghost=보더+표면. 트랜지션과 누름 효과.
- **차트**: 의존성 없는 **CSS 막대그래프**(그라데이션 fill). 리콜/불만/진단서.
- **마크다운**: 자체 경량 렌더러(##/###/불릿/**굵게**) — 진단서, 진단, Agent 답변.
- **빈 상태/가이드**: 업로드엔 예시 포맷 표 + 샘플 CSV 다운로드.

## 6. 왜 Tailwind/shadcn 안 썼나 (결정)

- 프로젝트가 이미 완성 단계 → **마이그레이션 리스크 > 실익**.
- 무의존 CSS로도 현대풍 미니멀은 충분히 구현 가능 + 빌드/배포 단순.
- 트레이드오프: 컴포넌트 재사용성은 shadcn이 우위. 규모 커지면 그때 전환 후보.

## 7. UX 카피 (라벨링 원칙)

"AI 같다"는 인상은 **영어 잔재, 중간점(·), 기능명 나열**에서 많이 온다. → **사용자 행동 중심 한국어**로 통일:

| 영역 | Before (AI틱) | After (자연스러움) |
|---|---|---|
| 네비/히어로 | RAG 검색 / Agent / Tabular SQL | 매뉴얼 검색 / 통합 질의 / 데이터 질의 |
| 카드 제목 | "Agentic Search (RAG+SQL 툴콜)" | "무엇이든 물어보기" |
| 카드 제목 | "이미지 진단 → 필요 부품 (Diagnose → Parts)" | "사진으로 진단받기" |
| 버튼/라벨 | Submit / Run / Answer / Sources | 질문 / 답변 / 근거 |
| 거버넌스 | Total Calls / Audit Trail / By Model | 총 호출 / 감사 로그 / 모델별 |

원칙:
- **영어와 괄호 병기 제거**(코드/식별자 제외), 중간점(·) 남발 금지.
- **기능명("text-to-SQL") 대신 행동("데이터로 질문하기")**.
- 기술 용어는 설명(히어로 desc와 hint)으로만, 제목은 사람 말로.

## 8. 접근성과 디테일

- 포커스 링(accent 18% glow), 충분한 대비, 키보드 Enter 제출.
- 한글 폰트 렌더링(antialiased, letter-spacing -.01em).
- 아이콘은 currentColor stroke로 테마 자동 대응.

## 9. 결과 표시 폴리시 (데이터 시각화)

데모에서 "결과가 안 와닿는다"는 인상을 줄이려고, 숫자에 **의미**를 색과 형태로 입혔다.

| 요소 | 처방 | 의도 |
|---|---|---|
| KPI 카드 | 화재>0 빨강, 부상>0 주황, 나머지 중립 | 위험 신호를 색으로 즉시 인지 |
| 점검표 결과 | 상태 배지(양호=초록, 교환=빨강, 정비=주황) | 표가 한눈에 읽힘 |
| 막대 차트 | 순위별 색 농도 램프(진한 네이비→연블루) | 1위가 가장 진하게, 위계 표현 |
| 거버넌스 | 모델별 호출을 텍스트 나열에서 막대 차트로 | 분포가 보임 |
| 상단바 | 백엔드 핑으로 `연결됨/꺼짐` 표시 | "진짜 제품" 신뢰감 |

빈 화면 처방: 결과가 없을 때 모든 탭에 **아이콘 + 한 줄 안내(무엇을 하는 탭인지)** 를 띄우고, 처리 중엔 회전 스피너. 의존성 없는 CSS만 사용(차트/배지/스피너 전부 CSS).

## 10. 다크모드 가시성 버그 (근본 원인과 수정)

증상: 다크모드에서 **색을 명시하지 않은 상속 텍스트**(표 셀, 문서명, 브랜드명, 감사 로그)가 안 보였다. 라벨이나 헤딩처럼 색을 직접 지정한 텍스트는 멀쩡.

근본 원인: 테마 토큰을 `[data-theme="dark"]`로 오버라이드하는데, 이 속성이 **`.shell`(콘텐츠 래퍼)** 에 붙는다. 그런데 `color: var(--text)`는 그 **상위인 `body`** 에만 걸려 있었다. `body`는 `.shell` 바깥이라 항상 `:root`의 어두운 `--text(#0d1b2a)`로 색이 계산되고, 색을 명시 안 한 자식들은 그 **계산된 어두운 색을 그대로 상속**받았다. (CSS 변수는 `.shell`에서 밝게 재정의됐지만, `color`를 다시 적용하는 요소만 그 값을 쓴다.)

수정: 테마 경계인 `.shell`에서 `color: var(--text)`를 **다시 적용**. 그러면 `.shell`의 계산된 색이 테마값(밝은색)이 되고, 모든 자식이 올바르게 상속한다. 한 줄로 전 화면 해결.

교훈: **CSS 변수 재정의 위치와 `color` 적용 위치가 어긋나면, 변수는 맞아도 상속 텍스트는 틀린다.** 테마는 색을 쓰는 노드와 같거나 그 위에서 적용해야 한다. 더불어 의미색(`--ok/--danger/--warn`)도 다크용 밝은 값을 따로 정의(라이트용 어두운 값은 다크 배경에서 안 보임).

## 11. 진단서 품질 — 결정적 집계와 LLM 서술의 분리

차량 진단 리포트는 **두 층**으로 나뉜다:

- **집계(KPI 카드·차트·점검표)** = 고정 SQL(결정적). 리콜/불만 건수, 상위 부품은 LLM을 안 거치고 DuckDB가 직접 센다 → 항상 정확, 재현 가능.
- **서술(종합 진단 문단)** = LLM. 보조 내러티브.

작은 파인튜닝 모델(1.5B)로 서술을 뽑으면 `S 모델`처럼 차종을 줄이고 `489건`·`7천 개` 같은 **숫자를 지어내며**, 끝이 잘리고 한자가 새는 문제가 있었다. 대응:

1. **프롬프트에 원시 자료구조를 넣지 않는다** — `List.toString()`이 `[[ENGINE, 1]]`로 새어 모델이 따라 뱉었다 → 사람이 읽는 문장으로 포맷팅.
2. **규칙 명시** — "주어진 수치만 인용, 대괄호·표 기호 금지, 4~6문장 완결".
3. **토큰 한도 상향**(256→512) — 문장 중간 잘림 방지.
4. **모델 라우팅** — 서술 품질이 중요한 진단서는 강한 모델(granite4)을 권장. 도메인 말투가 필요한 답변만 1.5B FT.

핵심 메시지: **신뢰가 필요한 수치는 결정적 경로(SQL)로, 자연어 서술만 LLM에 맡긴다.** 환각을 구조로 격리한 설계다.
